Projeto de Desenvolvimento Mobile

Sistema de Cadastro de Leitura e Vendas de Livros

  • Campus: Araguatins
  • Curso: Pós-Graduação Lato Sensu em Desenvolvimento de Sistemas Computacionais EaD
  • Disciplina: Desenvolvimento Mobile
  • Grupo: Eduardo Baranowski, Wesley Ferreira Sousa, Ramon Jordan Rubens e Robson Andrade Venceslau
  • Data: 01/07/2026

Contextualização

  • O mercado editorial e de leitura digital exige soluções mais organizadas e acessíveis.
  • Leitores precisam acompanhar livros, leituras e preferências em um ambiente único.
  • A proposta central é oferecer uma plataforma prática para conectar leitura, descoberta e interação.

Objetivo do Projeto

  • Desenvolver uma aplicação mobile com integração a um backend para apoiar o cadastro de leituras.
  • Facilitar a busca, organização e acompanhamento de livros pelos usuários.
  • Construir uma base para futuras expansões com vendas, recomendações e funcionalidades sociais.

Requisitos do Projeto

  • Cadastro e autenticação de usuários.
  • Registro de leituras e avaliações.
  • Busca e visualização de livros e editoras.
  • Gestão de catálogo e informações de livros.
  • Interface intuitiva e responsiva.

Metodologia Empregada

  • Levantamento de requisitos.
  • Definição da arquitetura cliente-servidor.
  • Desenvolvimento incremental com validações frequentes.
  • Testes e ajustes para maior estabilidade.

Ferramentas Empregadas

  • Flutter e Dart para o aplicativo mobile.
  • Node.js com TypeScript para o backend.
  • Prisma e PostgreSQL para persistência.
  • Swagger para documentação da API.
  • Git e GitHub para versionamento.

Benefícios da IA no Desenvolvimento

  • Aceleração na criação de estrutura e componentes.
  • Ajuda na geração de código e documentação.
  • Maior produtividade para a equipe.
  • Facilidade para explorar soluções e refatorações.

Dificuldades da IA no Desenvolvimento

  • Possibilidade de respostas incompletas ou incorretas.
  • Necessidade de validação manual.
  • Risco de dependência excessiva.
  • Limitações em contextos técnicos específicos.

Demonstração do Projeto

  • Tela de login e cadastro.
  • Catálogo e busca de livros.
  • Registro de leituras e atualização de status.
  • Estrutura preparada para expansão com vendas e funcionalidades sociais.

Considerações Finais

  • O projeto mostrou que é possível unir leitura, organização e tecnologia em uma solução prática.
  • O trabalho contribuiu para o aprendizado em desenvolvimento mobile, backend e integração.
  • A IA foi um apoio valioso, mas deve ser usada com critério e validação.

Trabalhos Futuros

  • Recomendações personalizadas de leitura.
  • Integração com vendas e pagamentos.
  • Notificações e comunicação entre usuários.
  • Expansão para relatórios e análise de hábitos de leitura.
