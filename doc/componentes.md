# Documentação de Componentes de Interface

## Componentes principais

### Bibliotheca
- Componente base para cabeçalhos, seções e cartões reutilizáveis.
- Centraliza estilos de espaçamento, tipografia e aparência visual.

### AppTheme
- Define a paleta, tipografia, raios e tema visual do app.
- Deve ser utilizado para manter consistência visual.

### Formulários de autenticação
- Login e cadastro devem seguir o mesmo padrão visual e comportamento de validação.
- Erros devem ser apresentados próximos aos campos e em blocos de feedback.

### Home / Feed
- Deve mostrar estado vazio, erro e carregamento de forma explícita.
- Ações de comentário e compartilhamento devem manter consistência visual.

### Detalhes do livro
- Deve indicar carregamento e falha de forma amigável.
- Ações de leitura e compra devem fornecer feedback claro.
